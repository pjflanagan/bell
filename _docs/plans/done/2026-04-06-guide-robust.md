
The guide is a very important part of this project. People who don't know about this project need to be able to 
learn about it before they install it and while they use it. 

Because of this, the guide needs to be robust. 
The guide needs to cover ALL features in the language. We need to add syntax examples that we do not have. 
All syntax examples should be pulled out into files (like examples) so they can be tested for correctness.

The examples also need to be improved. There are many comments that use `@` characters which is confusing.
In the docs, the files referenced by the examples should also be visible.

I also want multi file guides to be able to switch between files with tabs (as if they are in vscode).
